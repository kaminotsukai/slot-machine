# スロットマシンアプリ

TypeScriptで構築されたシンプルなスロットマシンアプリケーションです。3つの回転リール、勝敗判定、直感的なユーザーインターフェースを備えています。

## 機能

- 🎰 ランダムシンボル生成を備えた3リールスロットマシン
- 🎯 シンボルの組み合わせに基づく勝敗判定
- 🎨 クリーンで直感的なユーザーインターフェース
- 🔄 スムーズなスピンアニメーションと視覚的フィードバック
- 🧪 プロパティベーステストを含む包括的なテスト
- 📱 さまざまな画面サイズに対応したレスポンシブデザイン

## はじめに

### 前提条件

- Node.js (バージョン16以上)
- npm または yarn

### インストール

1. リポジトリをクローン
2. 依存関係をインストール:
   ```bash
   npm install
   ```

### 開発

- **プロジェクトをビルド**: `npm run build`
- **開発サーバーを起動**: `npm start` または `npm run dev`
- **テストを実行**: `npm test`
- **テストをウォッチモードで実行**: `npm run test:watch`
- **カバレッジレポートを生成**: `npm run test:coverage`

### アプリケーションの起動

開発サーバーを起動してブラウザでアプリケーションを開く:

```bash
npm start
```

ブラウザが自動的に開き、`http://localhost:8080` でアプリケーションが表示されます。

### プロジェクト構造

```
src/
├── types/           # コア型定義と列挙型
├── interfaces/      # コンポーネントインターフェース
├── StateManager.ts  # ゲーム状態管理
├── ReelManager.ts   # リール管理とシンボル生成
├── WinEvaluator.ts  # 勝敗判定
├── GameEngine.ts    # ゲームエンジン（コアロジック）
├── UserInterface.ts # ユーザーインターフェース
├── __tests__/       # テストファイル
└── index.ts         # メインエントリポイント
```

## アーキテクチャ

アプリケーションは、関心の明確な分離を持つModel-View-Controller（MVC）アーキテクチャに従っています：

- **GameEngine**: コアゲームロジックとオーケストレーション
- **ReelManager**: リールの回転とシンボル生成を処理
- **WinEvaluator**: 勝敗条件を判定
- **StateManager**: ゲーム状態の遷移を管理
- **UserInterface**: 視覚要素とユーザーインタラクションを処理

## 使い方

1. アプリケーションを起動すると、3つの空のリールとスピンボタンが表示されます
2. 「スピン」ボタンをクリックしてリールを回転させます
3. リールが停止すると、シンボルの組み合わせが表示されます
4. 3つのシンボルがすべて同じ場合、勝利メッセージが表示されます
5. 異なるシンボルの場合、次のスピンに進むことができます

## テスト

プロジェクトはデュアルテストアプローチを使用しています：

- **ユニットテスト**: Jestを使用した特定の例とエッジケース
- **プロパティベーステスト**: fast-checkを使用した普遍的なプロパティ

すべてのテストを実行:

```bash
npm test
```

テストカバレッジ:

```bash
npm run test:coverage
```
